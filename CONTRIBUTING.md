# Contributing to MyGreatLakes

We welcome all contributors who are passionate about improving the MyGreatLakes project.

## Contribution Guidelines

- Ensure your code follows our formatting and best practices.
- Write clear, concise commit messages.
- Include comments where necessary for clarity.
- Ensure all tests pass before submitting a pull request.

## How to Contribute

1. Fork the repository.
2. Clone your forked version locally.
3. Create a new branch for your feature/fix.
4. Make your changes.
5. Commit and push your branch.
6. Open a pull request to the `main` branch.

## Reporting Issues

- Use the Issues tab to report bugs or suggest improvements.
- Provide detailed steps to reproduce bugs.
- Include screenshots/logs when possible.

## Suggesting Features

- Open an issue with the `[Feature Request]` tag.
- Describe the feature and its purpose clearly.

## Code of Conduct

- Be respectful and inclusive.
- No harassment, discrimination, or offensive language.
- Constructive feedback is encouraged.

## Review Process

- All pull requests will be reviewed by maintainers.
- Automated checks must pass.
- Reviewers may suggest changes before approval.
